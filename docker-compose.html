Docker Compose cheatsheet

Краткая шпаргалка по Docker Compose с часто используемыми командами и конфигурациями.

#Начало работы

#Базовый цикл

$ docker compose up
$ docker compose up -d
$ docker compose down
  • up — запуск всех сервисов из docker-compose.yml
  • up -d — запуск в фоне (detached mode)
  • down — остановка и удаление сети/контейнеров проекта

#Файлы проекта

Файл Назначение
docker-compose.yml Основная конфигурация сервисов
docker-compose.override.yml Локальные переопределения (автоподхват)
.env Переменные окружения для compose-файла

#docker-compose.yml

#Минимальный пример

services:
  web:
    image: nginx:alpine
    ports:
      - "8080:80"
    volumes:
      - ./site:/usr/share/nginx/html:ro

  db:
    image: postgres:16
    environment:
      POSTGRES_DB: app
      POSTGRES_USER: app
      POSTGRES_PASSWORD: secret
    volumes:
      - pgdata:/var/lib/postgresql/data

volumes:
  pgdata:

#Ключевые секции

Секция Для чего
services Описывает контейнеры приложения
ports Проброс портов host:container
volumes Постоянное хранение данных
environment Переменные окружения сервиса
depends_on Порядок старта сервисов
networks Пользовательские сети между сервисами

#Управление сервисами

#Команды

Команда Описание
docker compose ps Список сервисов и их статус
docker compose logs -f Логи всех сервисов в реальном времени
docker compose logs -f web Логи конкретного сервиса
docker compose stop Остановить сервисы без удаления
docker compose start Запустить ранее остановленные сервисы
docker compose restart web Перезапустить конкретный сервис

#Выполнение команд

$ docker compose exec web sh
$ docker compose exec db psql -U app -d app
$ docker compose run --rm web npm test

exec выполняет команду в уже запущенном контейнере, run создаёт временный контейнер для одноразовой задачи.

#Сборка и публикация

#Образы

$ docker compose build
$ docker compose build --no-cache
$ docker compose pull
$ docker compose push
  • build — собрать образы из секции build: в compose-файле
  • --no-cache — игнорировать слой кэша при сборке
  • pull — скачать актуальные версии образов из registry
  • push — отправить собранные образы в registry

#Проверка конфигурации

$ docker compose config
$ docker compose config --services
$ docker compose config --volumes

Команда config разворачивает итоговый YAML и помогает проверить ошибки до запуска.

#Масштабирование и профили

#Масштабирование

$ docker compose up --scale web=3 -d
$ docker compose ps

Удобно для горизонтального масштабирования stateless-сервисов.

#Профили

services:
  mailhog:
    image: mailhog/mailhog
    profiles: ["dev"]

$ docker compose --profile dev up -d

#Очистка

#Очистка проекта

$ docker compose down
$ docker compose down -v
$ docker compose down --rmi local
  • -v — удалить тома проекта
  • --rmi local — удалить локально собранные образы

#Очистка Docker

$ docker system prune
$ docker volume prune
$ docker image prune -a
  • system prune — удалить неиспользуемые контейнеры/сети/кэш сборки
  • volume prune — удалить неиспользуемые тома
  • image prune -a — удалить неиспользуемые образы (включая без dangling-тега)

#Продвинутая конфигурация

#Healthcheck + depends_on

services:
  db:
    image: postgres:16
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U app"]
      interval: 5s
      timeout: 3s
      retries: 10

  api:
    build: .
    depends_on:
      db:
        condition: service_healthy

Это убирает гонку, когда API стартует раньше готовности базы.

#Переменные и env-файлы

services:
  web:
    env_file:
      - .env
      - .env.local
    environment:
      APP_ENV: production
      APP_PORT: "8080"
  • env_file — загрузить переменные из файла(ов)
  • environment — задать/переопределить переменные точечно
  • .env.local обычно добавляют в .gitignore для локальных секретов

#Secrets и configs

services:
  app:
    image: my-app:latest
    secrets: [db_password]
    configs: [app_config]

secrets:
  db_password:
    file: ./secrets/db_password.txt

configs:
  app_config:
    file: ./config/app.yml
  • secrets — передают чувствительные данные отдельно от переменных окружения
  • configs — удобно для неизменяемых конфигов, общих для сервисов
  • Оба объекта монтируются в контейнер как файлы

#Пересоздание и orphan-сервисы

$ docker compose up -d --build --force-recreate
$ docker compose down --remove-orphans
  • --build — пересобрать образы перед запуском
  • --force-recreate — пересоздать контейнеры даже без изменений конфигурации
  • --remove-orphans — удалить контейнеры сервисов, которых больше нет в compose

#Dev/Prod паттерны

#Несколько compose-файлов

$ docker compose -f compose.yml -f compose.dev.yml up -d
$ docker compose -f compose.yml -f compose.prod.yml up -d
  • -f — подключить дополнительный compose-файл
  • Параметры из второго файла перекрывают значения из первого
  • Удобно разделять dev/prod, не дублируя базовую конфигурацию

#Профили для локальных сервисов

$ docker compose --profile dev up -d
$ docker compose --profile tooling up -d

Отдельно включайте Mailhog, Adminer, Grafana и другие dev-инструменты.
