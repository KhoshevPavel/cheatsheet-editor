SSH cheatsheet

Краткая шпаргалка по SSH.

#Начало работы

#Подключение

Подключиться к серверу (порт 22)

$ ssh root@192.168.1.5

Подключение на указанный порт

$ ssh root@192.168.1.5 -p 6222

Подключение через pem-файл (права 0400)

$ ssh -i /path/file.pem root@192.168.1.5

См.: Права SSH

#Выполнение

Выполнение удалённой команды

$ ssh root@192.168.1.5 'ls -l'

Запуск локального скрипта

$ ssh root@192.168.1.5 bash < script.sh

Сжатие и загрузка с сервера

$ ssh root@192.168.1.5 "tar cvzf - ~/source" > output.tgz

#SCP

Копирование с удалённого на локальный

$ scp user@server:/dir/file.ext dest/

Копирование между двумя серверами

$ scp user@server:/file user@server:/dir

Копирование с локального на удалённый

$ scp dest/file.ext user@server:/dir

Копирование целой папки

$ scp -r user@server:/dir dest/

Копирование всех файлов из папки

$ scp user@server:/dir/* dest/

Копировать из папки сервера в текущую папку

$ scp user@server:/dir/* .

#Расположение конфигурации

Путь к файлу Описание
/etc/ssh/ssh_config Системная конфигурация
~/.ssh/config Пользовательская конфигурация
~/.ssh/id_{type} Приватный ключ
~/.ssh/id_{type}.pub Публичный ключ
~/.ssh/known_hosts Известные серверы
~/.ssh/authorized_keys Ключ для авторизованного входа

#Опции SCP

Параметры Описание
scp -r Rекурсивное копирование каталогов
scp -C Cжатие данных
scp -v Подробный vывод
scp -P 8080 Указать Pорт
scp -B Bатовый режим (без запроса пароля)
scp -p Сохранять время и режимы

#Пример конфигурации

Host server1
    HostName 192.168.1.5
    User root
    Port 22
    IdentityFile ~/.ssh/server1.key

Подключение по алиасу

$ ssh server1

См.: Полные опции конфигурации

#ProxyJump

$ ssh -J proxy_host1 remote_host2

$ ssh -J user@proxy_host1 user@remote_host2

Множественные прыжки (прокси)

$ ssh -J user@proxy_host1:port1,user@proxy_host2:port2 user@remote_host3

#ssh-copy-id

$ ssh-copy-id user@server

Копировать на сервер по алиасу

$ ssh-copy-id server1

Скопировать конкретный ключ

$ ssh-copy-id -i ~/.ssh/id_rsa.pub user@server

#Генерация ключей SSH

#ssh-keygen

$ ssh-keygen -t rsa -b 4096 -C "your@mail.com"
- - -
-t Тип ключа
-b Размер ключа в битах
-C Новый комментарий

Сгенерировать RSA 4096 бит с email в комментарии или использовать SSH Key Generator

#Генерировать

Интерактивная генерация ключа

$ ssh-keygen

Указать имя файла

$ ssh-keygen -f ~/.ssh/filename

Сгенерировать публичный ключ из приватного

$ ssh-keygen -y -f private.key > public.pub

Изменить комментарий

$ ssh-keygen -c -f ~/.ssh/id_rsa

Изменить пароль приватного ключа

$ ssh-keygen -p -f ~/.ssh/id_rsa

#Тип ключа

  • rsa
  • ed25519
  • dsa
  • ecdsa

#known_hosts

Поиск в known_hosts

$ ssh-keygen -F <ip/hostname>

Удалить из known_hosts

$ ssh-keygen -R <ip/hostname>

#Формат ключа

  • PEM
  • PKCS8

#SSH-туннелирование

#Параметры SSH-туннеля

Параметр Описание
-L Локальный проброс порта
-R Удалённый проброс порта
-D Динамический проброс порта
-f Запуск в фоне
-N Не выполнять удалённую команду
-g Разрешить удалённым хостам подключаться к проброшенным портам

Подробнее о флагах выше: man ssh

#Переадресация локального порта

# Пробросить локальный порт на удалённый сервер
ssh -L local_port:remote_host:remote_port user@ssh_server

# Пример: проброс локального порта 8080 на удалённый порт 80 на example.com
ssh -L 8080:example.com:80 user@ssh_server

#Динамическая переадресация портов

# Создать SOCKS-прокси на локальном порту
ssh -D local_port user@ssh_server

# Создать SOCKS-прокси на порту 1080 в фоне
ssh -f -N -D 1080 user@ssh_server

#Фоновый туннель

# Создать туннель с локальным пробросом порта в фоне
ssh -f -N -L local_port:remote_host:remote_port user@ssh_server

#Несколько туннелей

# Создать несколько туннелей в одной SSH-команде
ssh -L 8080:example.com:80 -L 3306:db.example.com:3306 user@ssh_server

#Объединение опций

# В фоне, без команды, разрешить удалённый хост, локальный проброс порта, identity_file
ssh -f -N -g -L 8080:example.com:80 -i ~/.ssh/custom_key user@ssh_server
