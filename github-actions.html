Github Actions cheatsheet

Краткая шпаргалка по GitHub Actions — инструменту автоматизации рабочих процессов: сборка, тестирование и развёртывание прямо из репозитория.

#Начало работы

#Введение

GitHub Actions— платформа CI/CD для автоматизации рабочих процессов: сборка, тестирование и развёртывание прямо из репозитория.

#Файлы рабочего процесса

Workflow GitHub Actions задаются в YAML-файлах, обычно в каталоге .github/workflows репозитория.

name: hello-world
on: push
jobs: hello-world-job: runs-on: ubuntu-latest steps: - name: Hello World run: echo "Hello World!"

Просмотр запусков workflow

  • На GitHub.com откройте главную страницу репозитория.
  • Под названием репозитория нажмитеActions.
  • В левой панели выберите нужный workflow (в примере — hello-world).

#Синтаксис рабочего процесса

name: learn-github-actions
run-name: ${{ github.actor }} is learning GitHub Actions
on: [push]
jobs: check-bats-version: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - uses: actions/setup-node@v3 with: node-version: '14' - run: npm install -g bats - run: bats -v

#Описание синтаксиса рабочего процесса

Линия Описание
name: Задаёт имя workflow. Метка для идентификации в репозитории.
run-name: Задаёт имя запуска с контекстом ${{ github.actor }} — пользователь, инициировавший запуск.
on: Событие, запускающее workflow. Здесь — любой push в репозиторий.
jobs: Определяет группу заданий workflow. Каждое задание выполняется независимо.
check-bats-version: Идентификатор задания в workflow. В примере — check-bats-version.
runs-on: Указывает тип машины для выполнения. Здесь — последняя версия Ubuntu.
steps: Содержит последовательность шагов, выполняемых в рамках задания.
uses: Подключает action к шагу. actions/checkout@v4 — клонирование репо, actions/setup-node@v3 — настройка Node.
with: Дополнительные параметры action. Используется вместе с uses.
node-version: Параметр under with — версия Node.js для setup-node. В примере — '14'.

#События

name: Event-trigger-on-push-example on: [push] # здесь определяется событие jobs: build: runs-on: ubuntu-latest steps: - uses: actions/checkout@v2 - name: Run a script run: echo "This workflow runs on every push to the repository."

#Задания (jobs)

Одно задание:

name: Single Job
on: [push]
jobs: build: runs-on: ubuntu-latest steps: - uses: actions/checkout@v2 - name: Run a build script run: script/build

Несколько заданий:

name: CI Workflow on: [push]

jobs: job-1: runs-on: ubuntu-latest steps: - uses: actions/checkout@v2 - name: Runs job 1 run: echo "Running Job 1" job-2: runs-on: ubuntu-latest steps: - uses: actions/checkout@v2 - name: Runs job 2 run: echo "Running Job 2" job-3: runs-on: ubuntu-latest steps: - uses: actions/checkout@v2 - name: Runs job 3 run: echo "Running Job 3"

#Шаги

jobs: build: runs-on: ubuntu-latest steps: # шаг 1 - name: Check out repository uses: actions/checkout@v2 # шаг 2 - name: Set up Node.js uses: actions/setup-node@v2 with: node-version: '14' # шаг 3 - name: Install dependencies run: npm install # шаг 4 - name: Run tests run: npm test

#GitHub Runners и self-hosted runners

Раннер GitHub:

name: Workflow
on: [push]
jobs: build: runs-on: ubuntu-latest # раннер по умолчанию steps: - uses: actions/checkout@v2 - name: Run a script run: echo "Hello, world!"

Собственный раннер

name: Workflow with Self-Hosted Runner
on: [push]
jobs: build: runs-on: self-hosted steps: - uses: actions/checkout@v2 - name: Run a script run: echo "Hello from self-hosted runner!"

#Переменные среды

Пользовательская переменная через env.

jobs: build: runs-on: ubuntu-latest env: CUSTOM_VARIABLE: 'Hello, World!' # Пользовательская переменная через env: steps: - name: Check environment variable run: echo "Value of CUSTOM_VAR is $CUSTOM_VAR"

#Секреты

Добавить секрет: Repository → Settings → Security → Secrets and Variables → Actions → New Repository Secret

Пример workflow с секретами:

name: Workflow with Secrets on: [push]

jobs: example_job: runs-on: ubuntu-latest steps: - name: Checkout repository uses: actions/checkout@v2 - name: Use a secret run: echo "The secret is ${{ secrets.MY_SECRET }}"

#Артефакты (artifacts)

Артефакты: Repository → Actions → Workflow Run → Artifacts

jobs: build: runs-on: ubuntu-latest steps: - name: Build project run: make build - name: Upload build artifact uses: actions/upload-artifact@v3 # готовый action для загрузки артефактов with: name: my-artifact path: path/to/artifact

#Кэширование зависимостей

Кэш зависимостей хранит загруженные пакеты и скомпилированные бинарники.

jobs: build: runs-on: ubuntu-latest steps: - uses: actions/checkout@v2 - name: Cache dependencies uses: actions/cache@v2 # хранит загруженные пакеты или скомпилированные бинарники with: path: |
            path/to/dependencies
            another/path
 key: ${{ runner.os }}-deps-${{ hashFiles('**/lockfile') }} # хеш lock-файла зависимостей генерируется в ОС - name: Install dependencies run: install-command

#Матричные стратегии (matrix)

jobs: build: runs-on: ubuntu-latest strategy: matrix: node-version: [12.x, 14.x, 16.x]
        # стратегия matrix позволяет запускать задачи в различных комбинациях окружений и ОС os: [ubuntu-latest, windows-latest, macOS-latest]
    steps: - uses: actions/checkout@v2 - name: Use Node.js ${{ matrix.node-version }} uses: actions/setup-node@v1 with: node-version: ${{ matrix.node-version }} - run: npm install - run: npm test env: CI: true

#Условия и выражения

Условия по ветке:

jobs: build: runs-on: ubuntu-latest steps: - name: Checkout code uses: actions/checkout@v2 - name: Run build if: github.ref == 'refs/heads/main' # Шаг "Run build" выполняется только если текущая ветка — main. run: make build

Условия по событию:

jobs: test: runs-on: ubuntu-latest steps: - name: Checkout code uses: actions/checkout@v2 - name: Run tests if: github.event_name == 'pull_request' # Шаг "Run tests" выполняется только по событию pull request run: npm test

#Команды рабочего процесса

На ubuntu-latest работают bash-команды.

steps: - name: Set environment variable run: echo "NAME=value" >> $GITHUB_ENV

#Конкурентность (concurrency)

Поле concurrency создаёт группу по github.head_ref. Новый запуск в той же группе отменяет текущие.

jobs: my_job: runs-on: ubuntu-latest concurrency: group: ${{ github.head_ref }} cancel-in-progress: true steps: - name: Run a script run: echo "Running script..."
