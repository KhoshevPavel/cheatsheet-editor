# cheatsheet-editor

Локальный редактор HTML-шпаргалок на базе одного файла [`cheatsheets/local-cheatsheet-editor.html`](cheatsheets/local-cheatsheet-editor.html).

## Требования

- **Браузер на Chromium** (Chrome, Edge, Brave и т.п.) с поддержкой [File System Access API](https://developer.mozilla.org/en-US/docs/Web/API/File_System_Access_API) — без него папку с записью на диск не открыть и **Сохранить** не сработает.
- **Доступ в интернет** при первой загрузке страницы: подключаются TinyMCE, CodeMirror и форматтер HTML с CDN (jsDelivr).

## Как открыть

Откройте `cheatsheets/local-cheatsheet-editor.html` в браузере **как локальный файл** (`file://`) или через любой статический сервер, если удобнее.

Стили визуального режима TinyMCE ссылаются на `../css/` относительно каталога `cheatsheets/`. Имеет смысл держать структуру репозитория как есть (рядом с `cheatsheets` лежит папка `css/`), чтобы превью в редакторе совпадало с внешним видом шпаргалок.

## Рабочий процесс

1. Нажмите **Открыть папку** и выдайте браузеру доступ **на чтение и запись** к каталогу, где лежат ваши `*.html` шпаргалки.
2. Слева появятся плитки всех HTML-файлов в **корне** выбранной папки (вложенные подкаталоги не обходятся). Сам файл `local-cheatsheet-editor.html` в список не попадает.
3. Выберите файл — откроются вкладки **Визуально** (TinyMCE) и **HTML** (редактор с подсветкой).
4. После правок нажмите **Сохранить** — содержимое записывается в выбранный файл на диске.

Дополнительно:

- **Обновить список** — пересканировать папку (например, после копирования файлов снаружи).
- **Поиск по названию или пути** — фильтр плиток слева.
- Удаление файла — кнопка ✕ на плитке (только при открытой папке с реальными файлами).

## Режим без открытой папки

До выбора каталога редактор может показать **встроенный список имён** типичных шпаргалок — это подсказка/заглушка. У таких плиток **нет дескриптора файла**: открыть и посмотреть шаблон можно, но **сохранить на диск** не получится, пока не нажмёте **Открыть папку** и не укажете каталог с реальными файлами.

## Визуальный редактор (TinyMCE)

- Обычное форматирование: списки, таблицы, ссылки, код и т.д.
- Кнопки раскладки для разметки шпаргалок: секции, сетки карточек (`h3`), колонки для блоков `h2` и т.п. Курсор должен быть **внутри** соответствующего блока (`.section`, `.h3-wrap-list`, `.h3-wrap`, `.h2-wrap`) — иначе редактор покажет предупреждение.
- У карточек с заголовком **h3** есть **ручка перетаскивания** — можно менять порядок блоков в пределах списка.

Содержимое полного HTML-документа синхронизируется с фрагментом вокруг `#mdLayout`, если он есть в файле.

## Вкладка HTML (источник)

- **Формат HTML** — выравнивание разметки (js-beautify).
- **Поиск** / **К тегу** — навигация по коду; в подсказке указаны горячие клавиши CodeMirror.

Горячие клавиши (в режиме HTML):

| Действие        | Клавиши        |
|-----------------|----------------|
| Сохранить       | Ctrl+S / Cmd+S |
| Поиск           | Ctrl+F / Cmd+F |
| Замена          | Ctrl+H / Cmd+H |
| Формат HTML     | Alt+F          |

## Что запоминается между сеансами

- **IndexedDB** — дескриптор открытой папки (после повторного захода можно быстро восстановить доступ через **Открыть папку**).
- **localStorage** — путь последнего редактируемого файла для автоматического открытия при следующем визите.

## Ограничения

- Нет рекурсивного обхода подпапок при сборе списка файлов.
- Нужен современный Chromium; Firefox/Safari без нужного API сохранение в файлы не дадут.
