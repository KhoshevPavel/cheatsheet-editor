Git cheatsheet

Шпаргалка с часто используемыми командами Git для быстрого доступа.

#Начало работы

#Создать репозиторий

Создать новый локальный репозиторий

$ git init [project name]

Клонировать репозиторий

$ git clone git_url

Клонировать репозиторий в указанную директорию

$ git clone git_url my_directory

#Создание коммита

Показать изменённые файлы в рабочей директории, подготовленные к коммиту

$ git status

Добавить файл в индекс, подготовить к коммиту

$ git add [file]

Добавить все изменённые файлы в индекс

$ git add .

Создать коммит со всеми файлами из индекса

$ git commit -m "commit message"

Создать коммит со всеми отслеживаемыми файлами

$ git commit -am "commit message"

Отменить изменения в рабочей директории (не в индексе)

$ git restore [file]

Убрать файл из индекса

$ git restore --staged [file]

Убрать файл из индекса, сохранив изменения

$ git reset [file]

Откатить всё к последнему коммиту

$ git reset --hard

Различия изменённого, но не добавленного в индекс

$ git diff

Различия добавленного в индекс, но ещё не закоммиченного

$ git diff --staged

Применить коммиты текущей ветки, опережающие указанную

$ git rebase [branch]

#Конфигурация

Установить имя для коммитов и тегов

$ git config --global user.name "name"

Установить email для коммитов и тегов

$ git config --global user.email "email"

Включить цветной вывод Git

$ git config --global color.ui auto

Открыть глобальный конфиг в редакторе

$ git config --global --edit

#Работа с ветками

Список всех локальных веток

$ git branch

Список всех веток (локальных и удалённых)

$ git branch -av

Переключиться на my_branch и обновить рабочую директорию

$ git checkout my_branch

Создать новую ветку new_branch

$ git checkout -b new_branch

Удалить ветку my_branch

$ git branch -d my_branch

Слить branchA в branchB

$ git checkout branchB
$ git merge branchA

Создать тег на текущем коммите

$ git tag my_tag

#Мониторинг репозитория

История коммитов текущей ветки

$ git log

Коммиты в branchA, которых нет в branchB

$ git log branchB..branchA

Коммиты, изменившие файл (включая переименования)

$ git log --follow [file]

Различия branchA относительно branchB

$ git diff branchB...branchA

Показать любой объект Git в читаемом формате

$ git show [SHA]

#Синхронизация

Загрузить все ветки с удалённого репозитория

$ git fetch [alias]

Слить удалённую ветку в текущую для обновления

$ git merge [alias]/[branch]
# No fast-forward
$ git merge --no-ff [alias]/[branch]
# Only fast-forward
$ git merge --ff-only [alias]/[branch]

Отправить коммиты локальной ветки в удалённый репозиторий

$ git push [alias] [branch]

Загрузить и слить коммиты из отслеживаемой удалённой ветки

$ git pull

Слить один конкретный коммит из другой ветки в текущую

$ git cherry-pick [commit_id]

#Удалённый репозиторий

Добавить URL git как алиас

$ git remote add [alias] [url]

Показать имена настроенных удалённых репозиториев

$ git remote

Показать имена и URL удалённых репозиториев

$ git remote -v

Удалить удалённый репозиторий

$ git remote rm [remote repo name]

Изменить URL git-репозитория

$ git remote set-url origin [git_url]

#Временные коммиты (stash)

Сохранить изменённые и добавленные в индекс файлы

$ git stash

Показать стек сохранённых изменений

$ git stash list

Применить изменения с вершины стека stash

$ git stash pop

Отменить изменения с вершины стека stash

$ git stash drop

#Отслеживание изменений путей

Удалить файл из проекта и добавить удаление в индекс

$ git rm [file]

Переименовать файл и добавить изменение в индекс

$ git mv [existing-path] [new-path]

Показать все коммиты с указанием переименованных путей

$ git log --stat -M

#Игнорирование файлов

/logs/*

# "!" means don't ignore
!logs/.gitkeep

/# Ignore Mac system files
.DS_store

# Ignore node_modules folder
node_modules

# Ignore SASS config files
.sass-cache

A .gitignore file specifies intentionally untracked files that Git should ignore

#Трюки Git

#Переименовать ветку

  • #Renamed to new_name

    $ git branch -m <new_name>

  • #Push and reset

    $ git push origin -u <new_name>

  • #Удалить remote branch

    $ git push origin --delete <old>

#Лог (log)

Поиск изменений по содержимому

$ git log -S'<a term in the source>'

Показать историю изменений конкретного файла

$ git log -p <file_name>

Вывести наглядную визуализацию лога

$ git log --pretty=oneline --graph --decorate --all

Список файлов, изменённых в коммите

$ git log --name-only <commit-id>

Список файлов, изменённых в коммитах (без слияний)

$ git log --no-merges --name-only

Ограничить коммиты последними N записями

$ git log -<number>

Ограничить коммиты последними N записями (альтернативный синтаксис)

$ git log -n <number>

Ограничить коммиты максимум N записями

$ git log --max-count=<number>

Показать коммиты новее указанной даты

$ git log --after="YYYY-MM-DD HH:MM:SS ±HHMM"

Показать коммиты старше указанной даты

$ git log --before="YYYY-MM-DD HH:MM:SS ±HHMM"

Фильтровать коммиты по автору (по шаблону)

$ git log --author=<pattern>

Фильтровать коммиты по сообщению (по шаблону)

$ git log --grep=<pattern>

Показать граф коммитов текущей ветки с декорациями ссылок

$ git log --graph --decorate

Показать граф коммитов указанной ветки без переключения на неё

$ git log --graph --decorate <branch>

Показать граф коммитов из одной ветки, отсутствующих в другой

$ git log --graph --decorate <branchB>..<branchA>

Показать граф коммитов всех веток

$ git log --graph --decorate --all

#Ветка (branch)

Список всех веток и их вышестоящих веток

$ git branch -vv

Быстро переключиться на предыдущую ветку

$ git checkout -

Получить только удалённые ветки

$ git branch -r

Восстановить один файл из другой ветки

$ git checkout <branch> -- <file>

#Перезапись истории

Переписать сообщение последнего коммита

$ git commit --amend -m "new message"

Изменить последний коммит без изменения сообщения.

$ git commit --amend --no-edit

#Продвинутый Git

#Рабочие деревья

Создать новое рабочее дерево по указанному пути. Если указана ветка — выполняется checkout этой ветки. Если ветка не указана — создаётся новая на основе HEAD с именем по имени новой директории.

$ git worktree add <path> [<branch>]

Список всех активных рабочих деревьев репозитория.

$ git worktree list

Удалить указанное рабочее дерево.

$ git worktree remove <path>

Очистить устаревшие или вручную удалённые рабочие деревья.

$ git worktree prune

Переместить рабочее дерево по новому пути.

$  git worktree move <old_path> <new_path>

#Субмодули (submodules)

Добавить новый субмодуль в репозиторий:

$ git submodule add <repository_url> <path>

Клонировать репозиторий и инициализировать его субмодули:

$ git clone --recursive <repository_url>

Обновить все субмодули до последнего коммита их веток:

$ git submodule update

Получить последние изменения из удалённых репозиториев субмодулей и обновить их в основном репозитории:

$ git submodule update --remote

Удалить субмодуль из репозитория:

$ git submodule deinit <path>
$ git rm <path>
$ git commit -m "Removed submodule"

#Cherry-pick

Cherry-pick позволяет применить конкретный коммит из одной ветки в другую.

$ git cherry-pick <commit_hash>

#Reflog

Показать reflog — историю перемещений HEAD и веток:

$ git reflog

Найти хеш потерянного коммита или ветки через reflog и выполнить checkout к этому хешу для восстановления:

$ git checkout <commit_or_branch_hash>
