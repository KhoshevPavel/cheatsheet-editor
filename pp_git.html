Git + GitHub cheatsheet

Шпаргалка по основным командам Git для повседневной работы от Pragmatic Programmer .

#1. Настройка и конфигурация

#Базовая конфигурация Git

Команда Описание
git config --global user.name "Имя"Установить имя пользователя
git config --global user.email "email"Установить email пользователя
git config --listПоказать все настройки
git config --global core.editor "editor"Установить редактор по умолчанию

#2. Создание и клонирование репозиториев

#Инициализация и клонирование

Команда Описание
git initСоздать новый локальный репозиторий
git init --bareСоздать bare-репозиторий (без рабочей директории)
git clone <url>Склонировать удаленный репозиторий
git clone <url> <dir>Склонировать репозиторий в указанную папку

#3. Основной workflow

#Работа с изменениями

Команда Описание
git statusПоказать состояние рабочей директории
git add <file>Добавить файл в индекс
git add .Добавить все изменения в индекс
git commit -m "Сообщение"Создать коммит
git commit --amendИсправить последний коммит

#Просмотр истории

Команда Описание
git logПоказать историю коммитов
git log --onelineКраткая история коммитов
git log --graphИстория с визуализацией веток
git show <commit>Показать изменения в коммите

#4. Ветки (Branches)

#Создание и переключение

Команда Описание
git branchПоказать список веток
git branch <name>Создать новую ветку
git checkout <branch>Переключиться на ветку
git checkout -b <branch>Создать и переключиться на ветку
git switch <branch>Переключиться на ветку (новый вариант)
git switch -c <branch>Создать и переключиться (новый вариант)

#Слияние и удаление

Команда Описание
git merge <branch>Слить ветку в текущую
git merge --abortОтменить слияние
git branch -d <branch>Удалить ветку
git branch -D <branch>Принудительно удалить ветку

#5. Работа с удаленными репозиториями

#Синхронизация с удалённым репозиторием

Команда Описание
git fetchЗагрузить изменения с удаленного репозитория
git pullЗагрузить и слить изменения
git pushОтправить изменения на удаленный репозиторий
git push -u origin <branch>Отправить и установить отслеживание
git push --forceПринудительная отправка (осторожно!)

#6. Отмена изменений

#Команды для отмены изменений

Команда Описание
git restore <file>Отменить изменения в файле
git restore --staged <file>Убрать файл из индекса
git reset --soft <commit>Отменить коммиты, сохранив изменения
git reset --hard <commit>Полностью отменить коммиты
git revert <commit>Создать коммит, отменяющий изменения

#7. Временное сохранение изменений

#Команды stash

Команда Описание
git stashВременно сохранить изменения
git stash listПоказать список сохраненных изменений
git stash popВосстановить последние изменения
git stash applyПрименить изменения без удаления
git stash dropУдалить сохраненные изменения

#8. Инспекция и сравнение

#Команды для анализа изменений

Команда Описание
git diffПоказать незакоммиченные изменения
git diff --stagedПоказать изменения в индексе
git diff <commit1> <commit2>Сравнить два коммита
git blame <file>Показать автора каждой строки файла
git grep "текст"Поиск текста в файлах репозитория

#9. Работа с GitHub

#Управление удалёнными репозиториями

Команда / действие Описание
git remote -vПоказать адреса удалённых репозиториев (fetch/push)
git remote add <name> <url>Добавить новый remote (обычно origin)
git remote set-url origin <new_url>Изменить URL удалённого репозитория
git remote remove <name>Удалить привязку к remote

#Pull Request / Merge Request workflow

Концепция / команда Описание
ForkКопия чужого репозитория в вашем аккаунте GitHub
git clone <url_fork>Клонировать свой форк локально
git remote add upstream <url_original>Добавить ссылку на оригинальный репозиторий
git fetch upstreamПолучить изменения из оригинала
git merge upstream/mainСинхронизировать свою ветку с оригиналом
Pull RequestЗапрос на вливание изменений через веб-интерфейс GitHub

#Доступ и безопасность

Команда / действие Описание
gh auth loginАвторизация в GitHub CLI
git push --delete origin <branch>Удалить ветку в удалённом репозитории
ssh -T git@github.comПроверить SSH-доступ к GitHub

#Практические советы по GitHub

  • Перед новой задачей синхронизуйте main: git pull origin main.

  • Используйте понятные имена веток: feature/..., bugfix/....

  • Связывайте коммиты с задачами: Fix #42 в сообщении коммита.

  • Включайте защиту веток и требуйте прохождение CI перед merge.

#10. Полезные команды для истории

#Расширенный поиск по истории

Команда Описание
git log --follow <file>История изменений файла с учетом переименований
git log -S "текст"Найти коммиты, где добавлялась/удалялась строка
git log --author=<pattern>Фильтр истории по автору
git log --grep=<pattern>Поиск коммитов по сообщению
git log --graph --decorate --allПолный граф веток и коммитов

#11. Перезапись истории и полезная диагностика

#Последние коммиты

Команда Описание
git commit --amend --no-editОбновить последний коммит без изменения сообщения
git commit --amend -m "текст"Изменить сообщение последнего коммита
git reset --mixed HEAD~1Убрать последний коммит, оставив изменения в файлах

#Восстановление через reflog

Команда Описание
git reflogПоказать историю перемещений HEAD
git reset --hard "HEAD@{1}"Откатиться к предыдущему состоянию HEAD
git show "HEAD@{1}"Посмотреть, что было в предыдущем состоянии

#12. Ускорение работы (aliases и файлы)

#Полезные алиасы

Команда Описание
git config --global alias.st statusКороткая команда для статуса
git config --global alias.co checkoutКороткая команда для checkout
git config --global alias.br branchКороткая команда для branch
git config --global alias.ci commitКороткая команда для commit

#Изменения файлов и путей

Команда Описание
git rm <file>Удалить файл и добавить удаление в индекс
git mv <old-path> <new-path>Переименовать или переместить файл
git log --stat -MПоказать историю с распознаванием переименований
git restore <file>Вернуть файл к состоянию последнего коммита

#Советы по использованию

#Перед коммитом

  • Всегда проверяйте git status перед коммитом.

  • Используйте git diff для просмотра изменений.

  • Делайте частые и атомарные коммиты.

#Хорошие привычки

  • Пишите осмысленные сообщения коммитов.

  • Перед слиянием всегда обновляйте свою ветку.

  • Держите историю чистой и понятной для команды.
